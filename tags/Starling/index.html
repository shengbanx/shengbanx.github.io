AS3 Starling

Starling学习笔记（二）

在官网可以知道，Starling是Sparrow 框架在ActionScript3领域的一个姐妹框架。基于stage3D的API开发，而Stage3D有能力去控制 OpenGL、DirectX这些电脑显卡驱动或OpenGL ES2这些手机显卡驱动，最终由这些显卡驱动程序去驱动GPU工作。也就是传说中的GPU显卡加速。

就有人会疑惑了，stage3D不是渲染3D内容的吗？为何用来做2D。这里用我们该如何使用Stage3D中的drawTriangles这个API来绘制类似MovieClip来解释。GPU的优势在于它绘制三角形极其高效，所以使用drawTriangles API将绘制出两个三角形，之后我们再从一个纹理（texture，你可以理解成“素材”的意思）上面采样（或取色）并使用一种称为UV映射*的机制来着色于这两个三角形上即可，最后我们将能够得到一个纹理化（textured，即着了色的）的方块，我们就用它来作为我们Sprite对象的外观。若是在每一帧都改变其纹理，我们就可以得到一个MovieClip对象了。

 

*UV 映射是一种纹理化对象的方法。它依赖于两个值：U 水平 (x) 值和 V 垂直 (y) 值。这两个值不是基于像素值，而是基于百分比。0 U 和 0 V 表示图像的坐上角，1 U 和 1 V 表示右下角  

那么，现在开始创建舞台元素玩玩了，如上一篇所讲，首先是创建启动类，然后在Game类中创建一个显示对象，建什么好呢？不妨查查API，starling.display，有Button、Image、MovieClip、Quad（四边形）以及其他一些容器。这里，我们创建一个button玩玩。

AS3 Starling

Starling学习笔记（一）

开学零零碎碎的时间，两个星期了，主要在学习Starling框架，给我的感觉很不错，易用，效率高。

刚开始还无从下手，都是在Starling中文站照着学习和查阅API文档（中文）。还有就是中文说明书，学着学着就大概知道怎么用了。

Starling框架让使用Stage3D技术(在flash player 11中引入)创建二维程序和游戏成为可能。 它实现了和Flash 一样的显示列表系统，同时使用先进的GPUs加速渲染。以下讲讲我的体会：

开发工具：Flash builder4.6(FB4.7用不习惯）, Flash Develop。（给我的感觉是fd写代码更快更方便一点）

Starling框架：到官网下载starling包，里面有starling库源码和swc库文件以及一些demo项目文件。

Starling第一步

将您的项目链接到Starling swc库：

FB：建立新项目，在项目里建一个libs文件夹，把swc文件拉到里面，然后项目-属性-构建路径里添加libs（swc文件夹/文件）。

FD：建立新项目，把swc文件拉到libs文件夹里，然后右键选择-添加到库。

建立启动类：

src源路径里添加新类（class）文件——作为starling启动类，继承自flash的Sprite（而下面Game类以及之后用到starling的都继承自starling的sprite）。代码如下：

package
{
    import flash.display.Sprite;
    import starling.core.Starling;


[SWF(frameRate = 60, width = 600, height = 1024, backgroundColor = 0xffffff)]
public class Startup extends Sprite
{    

    public function Startup()
    {
        var _starling:Starling = new Starling(Game, stage);
        _starling.start();
    }

}

}
上面用核心starling类创建了starling对象_starling，然后用start方法启动它，这样，他就会按照设置好的信息[SWF...]渲染Game类了，Display显示列表和as原本的名字基本相似，所以容易混淆（常常自动填写代码错误...），但starling使用了stage3D，运行速度快。starling类还有其他用处，例如我常常就会习惯的加上_starling.showStats = true;来显示一些FPS和内存占用信息。
