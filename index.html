Morn UI air

Morn UI使用感受

在开发AIR桌面应用GIMS的时候，想起之前在微博上的看到的MornUI，于是便开始了接触了这个有爱的Morn UI~

Morn UI简介

Morn UI,是一个flash轻量级，高性能，可视化UI解决方案。它包括了一个Morn UI库和一个搭配的可视化编辑器Morn Builder，下面是官方介绍的优势,基本上都认同：

  1. 轻量级，Morn UI库以精简，直观为设计理念，代码轻量，能快速上手，减少学习成本，Morn UI全部库总大小不到30K
  2. 可视化，Morn UI提供强大的可视化编辑器，布局及属性均可在编辑器直观设置，实现UI和逻辑分离，让美术和程序轻松合作
  3. 高性能，Morn UI设计之初就以高性能为主要目标，以位图为基础，利用延迟渲染机制，实现了高性能
  4. 易扩展，无论UI组件还是编辑器插件，都非常易于扩展，编辑器支持即改即用，轻松实现个性化
  5. 自动资源管理，Morn编辑器自动管理UI资源，以灵活的方式打包资源（jpg+alpha）
  6. 同时支持位图和矢量图，并自动生成UI代码
  7. 支持多项目，支持多视图开发，支持代码视图，支持协同开发及svn管理
  8. 支持多语言，内置语言包管理器及提取工具，快速国际化
  9. Morn比flex轻量高效，比flash

新版本已经全面支持移动设备开发了，其作者yungzhu是个热心人（在q群上~），在官博上有demo和使用教程，还有在线API文档大概一个下午就能摸熟，上手不难，但一经上手，你就会深深地爱上了它！（这是所有第一次使用的人的共识，但因为Morn UI还在不断完善，所以随着深入开发你就会发现其坑之多，且陷入其中，“其乐融融”！）

ActionScript3 Event Flash

Actionscript3事件调度与事件流

在做pulupulu游戏时，我再次研究了下，以前一直没认真搞懂的事件流（Event Flow）。发现以前真的很无知，对于事件（Event）。

首先，AS3中，事件可分为两类：内建事件，它描述运行期环境的状态变化；自定义事件，它描述一个程序的状态变化。内建事件包括用户输入、点击鼠标等事件，而自定义事件则更广泛，可能是一个游戏的开始或一次文本的提交。

为了处理（Handle)或响应事件，使用事件侦听器（Event Listener）。事件侦听器是一个函数或方法，它注册是为了等待一个事件发生时执行。当一个给定的事件已经发生时为通知一个程序，as执行所有已经注册了该事件的事件侦听器，这个通知过程称为一个事件调度（Event DIspatch）。

当as以一个对象为目标调度一个事件时，如果该对象不是一个显示层次结构的一部分，则那个目标是被通知该事件的唯一对象。相对而言，如果该对象是一个显示层次结构的一部分，则事件对象将会在这个显示列表中“穿行”，直到抵达事件目标，它不仅通知了那个目标同时也通知了一个显示列表的对象。这便是形象的事件流（Event Flow）。

如上所言，事件调度分为三个阶段：捕获阶段（Capture Phase），目标阶段（Target Phase），冒泡阶段（Bubbling Phase）。

捕获阶段，目标的同显示列表里的元素都会被通知该事件已发生。这是一个向下探的过程，在探到目标之前路过其他东西都会被摸到。

目标阶段，as通知目标对象该事件发生了。

对于一些事件*，事件调度到此就结束了。但对于其他事件，将继续前进到第三阶段——

冒泡阶段，目标的同显示列表的对象被通知该目标已成功接收到事件通知。

似乎只需要捕获及目标阶段就可以了，但既然存在三个阶段，当然是为了适应更多的开发需求。

三个阶段就如捕鱼的——撒网、捕捉、收网~不同的是这里撒网只为捕捉一条鱼。

值得注意的是框架Starling的事件没有“捕获”的阶段。这些事件只是简单的被一个对象派发，并可以选择 冒泡。他们不能向相反的方向传递。当派发事件（dispatchEvent）时，即派发事件到所有注册了这个事件类型侦听器的对象。如果一个事件开启了"冒泡"，它会沿着父级关系，一直向上，直到碰到了根节点，或者中间被手动停止它的传递。

*四种事件类型只有一个目标阶段——Event.ACTIVATE、Event.DEACCTIVATE、Event.ENTER_FRAME、Event.RENDER。 参考：

AS3 Starling

Starling学习笔记（二）

在官网可以知道，Starling是Sparrow 框架在ActionScript3领域的一个姐妹框架。基于stage3D的API开发，而Stage3D有能力去控制 OpenGL、DirectX这些电脑显卡驱动或OpenGL ES2这些手机显卡驱动，最终由这些显卡驱动程序去驱动GPU工作。也就是传说中的GPU显卡加速。

就有人会疑惑了，stage3D不是渲染3D内容的吗？为何用来做2D。这里用我们该如何使用Stage3D中的drawTriangles这个API来绘制类似MovieClip来解释。GPU的优势在于它绘制三角形极其高效，所以使用drawTriangles API将绘制出两个三角形，之后我们再从一个纹理（texture，你可以理解成“素材”的意思）上面采样（或取色）并使用一种称为UV映射*的机制来着色于这两个三角形上即可，最后我们将能够得到一个纹理化（textured，即着了色的）的方块，我们就用它来作为我们Sprite对象的外观。若是在每一帧都改变其纹理，我们就可以得到一个MovieClip对象了。

 

*UV 映射是一种纹理化对象的方法。它依赖于两个值：U 水平 (x) 值和 V 垂直 (y) 值。这两个值不是基于像素值，而是基于百分比。0 U 和 0 V 表示图像的坐上角，1 U 和 1 V 表示右下角  

那么，现在开始创建舞台元素玩玩了，如上一篇所讲，首先是创建启动类，然后在Game类中创建一个显示对象，建什么好呢？不妨查查API，starling.display，有Button、Image、MovieClip、Quad（四边形）以及其他一些容器。这里，我们创建一个button玩玩。

Button:

一个简单的按钮由一张图片和一个可选的文本组成。

你可以分别为按钮构造函数传递up和down两种状态的纹理。 如果你没有指定一个down状态的纹理，按钮会采用默认的办法：被触碰的时候缩小一点。 此外，你可以在按钮上覆盖一个文本，要自定义这个文本，只需要提供和原来的基本相同的文本框即可。 你可以使用textBounds属性，将文本移动到一个指定的位置。

判断按钮是否被触碰，请使用triggered事件类型， 用这个事件来代替普通的触碰事件。也就是说，用户只要在松开手指或者鼠标之前，移动手指/鼠标离开按钮区域，就可以取消对该按钮的点击操作。  

package
{
    import starling.display.Button;
    import starling.display.Sprite;
    import starling.textures.Texture;


/**
 * ...
 * @author SHENGbanX
 */
public class Game7 extends Sprite
{
    [Embed(source = "../assets/chicken.png")]
    private static const Chicken:Class;
    public function Game7()
    {
                    //通过Texture类中的静态方法fromBitmap来创建Texture纹理对象
        var buttonTexture:Texture = Texture.fromBitmap(new Chicken());
        var button:Button = new Button(buttonTexture, "Hello Starling");
        button.x = 200;
        button.y = 200;
                    addChild(button);
    }

}

} 自己用ps简单地画了一个小鸡作为按钮的图片纹理，加上文字。但这样文字刚好在图片上，所以要用textBounds属性，把文字移到一个地方。

查api可知textBounds:Rectangle，Rectangle是flash.geom包（包中包含几何图形类（如点、矩形和转换矩阵）以支持 BitmapData 类和位图缓存功能。）里的一个类。

而Rectangle 对象是按其位置（由它左上角的点 (x, y) 确定）以及宽度和高度定义的区域。

加上button.textBounds = (0,50,100,100);

但报错了，类型不对，Rectangle不是就4个数作属性吗？思来想去，还是去看看starling的源代码：

        /* The bounds of the textfield on the button. Allows moving the text to a custom position. /
        public function get textBounds():Rectangle { return mTextBounds.clone(); }
        public function set textBounds(value:Rectangle):void
        {
            mTextBounds = value.clone();
            createTextField();
        }
仔细研究了一下，发现还是应该是那样没错的，试了很多种方法，通过trace(button.textBounds)结果是(x=0, y=0, w=73, h=53),以这个做值或(x=0, y=0, width=73, height=53)也不行。

最后这样才可以：

button.textBounds = new Rectangle(0, 50, 73,53);
 

没有教程的时候，查API、源码等方法自己研究还是很不错的！学习Starling更需如此。

AS3 Starling

Starling学习笔记（一）

开学零零碎碎的时间，两个星期了，主要在学习Starling框架，给我的感觉很不错，易用，效率高。

刚开始还无从下手，都是在Starling中文站照着学习和查阅API文档（中文）。还有就是中文说明书，学着学着就大概知道怎么用了。

Starling框架让使用Stage3D技术(在flash player 11中引入)创建二维程序和游戏成为可能。 它实现了和Flash 一样的显示列表系统，同时使用先进的GPUs加速渲染。以下讲讲我的体会：

开发工具：Flash builder4.6(FB4.7用不习惯）, Flash Develop。（给我的感觉是fd写代码更快更方便一点）

Starling框架：到官网下载starling包，里面有starling库源码和swc库文件以及一些demo项目文件。

Starling第一步

将您的项目链接到Starling swc库：

FB：建立新项目，在项目里建一个libs文件夹，把swc文件拉到里面，然后项目-属性-构建路径里添加libs（swc文件夹/文件）。

FD：建立新项目，把swc文件拉到libs文件夹里，然后右键选择-添加到库。

建立启动类：

src源路径里添加新类（class）文件——作为starling启动类，继承自flash的Sprite（而下面Game类以及之后用到starling的都继承自starling的sprite）。代码如下：

package
{
    import flash.display.Sprite;
    import starling.core.Starling;


[SWF(frameRate = 60, width = 600, height = 1024, backgroundColor = 0xffffff)]
public class Startup extends Sprite
{    

    public function Startup()
    {
        var _starling:Starling = new Starling(Game, stage);
        _starling.start();
    }

}

}
上面用核心starling类创建了starling对象_starling，然后用start方法启动它，这样，他就会按照设置好的信息[SWF...]渲染Game类了，Display显示列表和as原本的名字基本相似，所以容易混淆（常常自动填写代码错误...），但starling使用了stage3D，运行速度快。starling类还有其他用处，例如我常常就会习惯的加上_starling.showStats = true;来显示一些FPS和内存占用信息。

查API文档可以知道，starling类还可以：

访问Starling对象

在你的程序里，你可以在任意时间通过静态方法Starling.current访问当前的Starling对象。它会返回 当前活跃的Starling实例（实际上大多数程序只会有一个Starling对象）。

观察口，视口

即是Starling内容渲染显示的范围，默认情况下是舞台大小。但是，你可以使用“viewport”属性去改变它。当你只想在屏幕的一部分中 渲染，或者播放器大小改变时，这个属性非常有用。随后你可以监听由Starling舞台发出的RESIZE事件。

原生Flash叠加层

有时你会想在Starling上面显示一些原生Flash内容，这就是nativeOverlay属性的作用。它返回的是在Starling之上 的原生FLash Sprite。你可以在叠加层内添加原生Flash对象。

但是需要注意，在3D加速内容之上的原生Flash内容在某些（移动）平台上会有性能上的问题。因为这个原因，请注意删除这一层中所有 不需要和不再使用的对象。当这一层被置空后Starling会从显示列表中删除原生Flash叠加层。

多点触摸

对于提供支持多点触摸功能的设备，Starling也支持这项功能。在开发过程中，我们大多数基本上都是使用鼠标的键盘，Starling可以 在“Shift”和“Ctrl”的帮助下（Mac苹果机是“Cmd”）模拟实现多点触摸事件。启动simulateMultitouch属性就可以激活 这个功能。

处理丢失的渲染内容

在某些系统和某些条件下（例如，从系统睡眠中返回），Starling的stage3D渲染内容会丢失。如果类属性“handleLostContext” 设置为 “true” Starling会自动修复丢失内容。但是需要注意，这是以提高内存消耗为代价的。Starling会在内存中创建纹理缓存，这样 在内容丢失时才可以修复它。

如果你想和丢失的内容交互，Starling在这个内容修复时发送"Event.CONTEXT3D_CREATE"事件，你可以在相对应的事件*中重新 创建这些不可用资源.

共享三维加速内容

正常情况下，Starling是独立处理stage3d加速内容。如果你想让Starling和其它的stage3d引擎同时使用，可能会达不到你需要的效果。 在这种情况下，你可以使用shareContext属性:

  1. 手动创建和定义一个让两个框架公用的context3D对象 (通过stage3D.requestContext3D 和 context.configureBackBuffer)创建.
  2. 使用已经定义了的context3d对象的stage3d实例来初始化Starling。 这样做会自动启用 shareContext.
  3. 调用Starling实例的 start()(与往常一样). 这样做是让 Starling 对事件监听进行排队处理 (keyboard/mouse/touch).
  4. 创建一个游戏逻辑（例如，使用原生的ENTER_FRAME事件）然后让它调用Starling的 nextFrame同时调用其它Stage3D引擎的同功能方法 . 相关类型的方法还有context.clear() 和 context.present().  

接着新建一个Game类，

import starling.display.Sprite;
import starling.text.TextField;


public class Game extends Sprite
{
    public function Game()
    {
        var textField:TextField = new TextField(400, 300, "Hello Starling!");
        addChild(textField);
    }
}
