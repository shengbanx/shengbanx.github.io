ActionScript3 AS3 一串小球 无聊小球

一个小球到一串小球……@无聊小球

是这样的，又看书学着做了一个小球弹性运动，就是一条橡皮筋拉着小球运动，橡皮筋的另一头就是你的鼠标位置。 scale3.as代码如下：

package  {
    import oneball.Ball;
    import flash.display.Sprite;
    import flash.events.Event;


public class Scale3 extends Sprite{

    //Mouse.hide();

    private var friction:Number=0.85;
    private var g:Number=0.98;
    private var mass:Number=10;
    private var gravity:Number=g*mass;
    private var ball:Ball;
    private var scale:Number=0.3;
    private var vx:Number=50;
    private var vy:Number=50;

    public function Scale3() {
        init();
    }

    private function init():void{
        ball=new Ball;
        addChild(ball);
        addEventListener(Event.ENTER_FRAME,onEnterFrame);
    }

    function onEnterFrame(e:Event):void{
        var dx:Number=mouseX-ball.x;
        var dy:Number=mouseY-ball.y;
        var ax:Number=dx*scale;
        var ay:Number=dy*scale;
        vx+=ax;
        vy+=ay;
        vy+=gravity;
        vx*=friction;
        vy*=friction;
        ball.x+=vx;
        ball.y+=vy;
        graphics.clear();
        graphics.lineStyle(1);
        graphics.moveTo(ball.x,ball.y);
        graphics.lineTo(mouseX,mouseY);
    }

}

}
然后，由一个小球变成一串小球的时间到了！总不会是一个个代码添加吧，肯定不会啦！在之前的ball类加入自定公用变量属性vx和vy，并且把初始值改为0。使用moveBall方法，包含所有运动代码。然后创建一个数组来保存对这串球的每一个小球的引用，然后循环整个数组来运动每一个小球，然后就绘制直线。

具体就是，开始时上面init函数通过for循环创建每一个物体，并添加到数组中，然后onEnterFrame方法中，开始时设置直线样式，移动绘制的位置到鼠标位置，移动第一个小球，绘制一个到它的直线。，然后循环数组中剩下的小球，轮流移动并接着绘制线。

Scale4代码如下：

package  {
    import oneball.Ball;
    import flash.display.Sprite;
    import flash.events.Event;


public class Scale4 extends Sprite{

    //Mouse.hide();
    private var balls:Array;
    private var numBalls:Number=5;
    private var friction:Number=0.7;
    private var g:Number=0.98;
    private var mass:Number=10;
    private var ball:Ball;
    private var gravity:Number=g*mass;
    private var scale:Number=0.4;
    private var vx:Number;
    private var vy:Number;

    public function Scale4() {
        init();
    }

    //for循环创建小球
    private function init():void{
        balls=new Array();
        for(var i:uint=0;i&lt;numBalls;i++){
            var ball:Ball=new Ball(20);
            addChild(ball);
            balls.push(ball);
        }
        addEventListener(Event.ENTER_FRAME,onEnterFrame);
    }

    function onEnterFrame(e:Event):void{
        graphics.clear();
        graphics.lineStyle(1);
        graphics.moveTo(mouseX,mouseY);
        moveBall(balls[0],mouseX,mouseY);
        graphics.lineTo(balls[0].x,balls[0].y);

        for(var i:uint=1;i&lt;numBalls;i++){
            var ballA:Ball=balls[i-1];
            var ballB:Ball=balls[i];
            moveBall(ballB,ballA.x,ballA.y);
            graphics.lineTo(ballB.x,ballB.y);
        }
    }
    private function moveBall(ball:Ball,targetx:Number,targety:Number):void{
        ball.vx+=(targetx-ball.x)*scale;
        ball.vy+=(targety-ball.y)*scale;
        ball.vy+=gravity;
        ball.vx*=friction;
        ball.vy*=friction;
        ball.x+=ball.vx;
        ball.y+=ball.vy;
    }

}

}
Ball类代码Ball.as：

package{
    import flash.display.Sprite;


public class Ball extends Sprite{
    public var radius:Number;
    private var color:uint;
    public var vx:Number=0;
    public var vy:Number=0;

    public function Ball(radius:Number=40,color:uint=0x116666){
        this.radius=radius;
        this.color=color;
        init();
    }
    public function init():void{
        graphics.beginFill(color);
        graphics.drawCircle(0,0,radius);
        graphics.endFill();
    }
}

}
可以看到，只要修该numBall的值就可任意添加小球，修改“摩擦系数”和“弹力系数”和或者去掉重力，不同组合有不同玩法，绝对玩不厌，还可以调高帧频，看起来更快。玩法多样，如：10个小球组成的毛毛虫，改小摩擦系数0.6，或者还可以去掉重力，然后用鼠标模拟毛毛虫运动吧.... 真的好无聊，还可以把1000个圆叠加成“画笔”来画…….

无聊加条线惹的祸……@无聊小球

这几天经常没事就做做小球运动，今天学着书做了个小球缓动过程，也就是:

(1)设定一个比例系数（<1）；

(2)设定目标点；

(3)计算物体到目标点之间的距离；

(4)将距离与比例系数相乘，得到速度；

(5)加入速度到当前物体的位置；

(6)重复3-5步； 那么这小段代码便是：

下面是改完后的Scale2.as完整代码：

package {
    import oneball.Ball;
    import flash.events.Event;
    import flash.display.Sprite;
    import flash.events.MouseEvent;
    import flash.ui.Mouse;


public class Scale2 extends Sprite{
    private var ball:Ball;
    private var scale:Number=0.2;
    private var targetx:Number=stage.stageWidth/2;
    private var targety:Number=stage.stageHeight/2;
    private var vx:Number;
    private var vy:Number;

    //添加小球
    public function Scale2() {
        init();
    }

    private function init():void{
        ball=new Ball();
        addChild(ball);
        ball.addEventListener(MouseEvent.MOUSE_DOWN,onMouseDown);
        addEventListener(Event.ENTER_FRAME,onEnterFrame);
    }

    //鼠标拖动
    //鼠标按下
    private function onMouseDown(event:MouseEvent):void{
        ball.startDrag();
        stage.addEventListener(MouseEvent.MOUSE_UP,onMouseUp);
        stage.addEventListener(MouseEvent.MOUSE_MOVE,onMouseMove);
        removeEventListener(Event.ENTER_FRAME,onEnterFrame);
    }

    //鼠标移动
    private function onMouseMove(event:MouseEvent):void{
        addEventListener(Event.ENTER_FRAME,addline);
        function addline(e:Event):void{
            graphics.clear();
            graphics.moveTo(targetx,targety);
            graphics.lineStyle(1);
            graphics.lineTo(ball.x,ball.y);
        }
    }

    //松开鼠标
    private function onMouseUp(event:MouseEvent):void{
        ball.stopDrag();
        addEventListener(Event.ENTER_FRAME,onEnterFrame);
        stage.removeEventListener(MouseEvent.MOUSE_UP,onMouseUp);
        stage.removeEventListener(MouseEvent.MOUSE_MOVE,onMouseMove);
    }

    //重复计算速度--缓动(增加停止条件）
    function onEnterFrame(event:Event):void{
        var dx:Number=targetx-ball.x;
        var dy:Number=targety-ball.x;

        //停止缓动
        if(Math.abs(dx)<1||Math.abs(dy)<1){
            ball.x=targetx;
            ball.y=targety;
            removeEventListener(Event.ENTER_FRAME,onEnterFrame);
            trace("停止“缓动计算”成功")
        }else{
            graphics.clear();
            vx=(targetx-ball.x)*easing;
            vy=(targety-ball.y)*easing;
            ball.x+=vx;
            ball.y+=vy;

            //无聊加条线
            graphics.moveTo(targetx,targety);
            graphics.lineStyle(1);
            graphics.lineTo(ball.x,ball.y);
        }
    }
}

}

发展方向 应用领域 谈Flash

了解flash应用领域和发展方向

说真的，以前根本没有想过flash的用处与地位这么重要，在之前一段时间看一些视频制作高手用其他软件制作的片头，和一些CG动画感觉flash制作的动画、视频真的一无是处，既浪费时间，画面也不怎么样。有时也想过是不是要放弃flash而学习其他软件（因为我那时想着flash用来做动画）。当时我意识里，flash就只是用来做动画、视频和一些游戏应用等。但是我as3.0学了很久也学不好，所以做游戏暂时没想。但是自这次深入了解flash的应用领域、发展方向后，发现flash原来是真的很强大，flash也有其优势的方面，具有跨平台、体积小、运行效率高、高灵活性、应用范围广等许多特点，由于其“灵活”，所以其发展方向也出奇的多，很多都是很难想象的，我也开始知道了国内外众多flasher的努力，让flash在众多领域都发挥出其出众的效果，虽然有些只是提出些思想与一些实例，例如生物技术的模拟（人工智能）、压缩技术的开发，但其灵感就能让其他flasher借鉴与努力。

在我看完一篇篇讲flash的的文章之后，我也开始深深地喜欢上了flash，开始认真对待flash，做一个真正意义上得flasher。下面是引用其中一篇的一段。

那么，flash可能的发展方向究竟如何？ 我不能说这些方向中的所有都会实现，因为这也取决于Macromedia的决心，而更多的是flasher们的努力。

应用程序：由于其独特的跨平台物性和在界面控制以及多媒体的功能使用得使用flash来制作的应用程序具有很强的生命力。在与用户的交流方面具有其它任何方式都无可比拟的优势。当然，某些功能你可能还要依赖于XML或者其它诸如javascript的客户端技术来实现，但你能方便在它们的实现数据通信。 它所面临的问题： 没有经验，事实上，很少有人具有运用flash进行应用程序开发这方面的经验。但这个难度会随着时间的推移而逐步减弱。事实上，对于你的大型项目而言。使用flash此时未免有些言之过早，因为它意味着很大的风险。当然，在最早的时间掌握和积累这方面的经验无疑是一种很大的竞争力，这对于而言是一种很大的诱惑。代替可能的风险，你可以只将它运用在项目中的一小部份或者小型项目中。并且注意你的用户反馈意见。

操作系统界面：Flash对于界面元素的可控性和它所有表达的效果无疑具有很大的诱惑。对于一个系统的界面你可能要花更多的时间关注它们的接口和界面显示能力间的驱动。另外一个需要关注的就是你的内存使用和执行效率。

手机领域的开发：转向手机领域的开发将会要求你对精确（像素级）的界面设计和CPU的使用分布的操控能力更高。但也意味更大更广泛的使用空间。事实上手机和Pocket PC的分界已越来越不明显，你必须为每一款手机（或Pocket PC）设计一个不同的界面，因为它们的屏幕大小各有不同。当然你的内核可能是相同的，所要注意的是你的各类手机的CPU的计算能力和内存的大小。这无疑是些很苛刻的要求。

游戏开发：事实上，Flash中的游戏开发已经进行了多年的尝试。但至今为止仍然停留在中、小型游戏的开发上。游戏开发的很大一部份都受限于它的CPU能力和大量代码的管理。不过可喜的是，Flash Player 7 运行时性能提高了 2 至 5 倍；而且基于 Java 类的结构的类的开始也奠定了这些基础。

Web应用服务：很难界定Web应用服务的范围究竟有多大。似乎它拥有无限的可能。随着网络的逐渐渗透，基于客户端-服务器的应用设计也开始逐渐受到欢迎，并且一度被誉为最具前景的方式。这种方式你可能要花更多的时间在服务器的后台处理能力和架构上，并且将它们与前台（Flash端）保持同步。

站点建设：事实上，现在只有极少数人掌握了使用Flash建立全flash站点的技术。因为它意味着更高的界面维护能力和你的整站的架构能力。但它带来的好处也异常明显：全面的控制；无缝的导向跳转；更丰富的媒体内容；更体贴用户的流畅交互；以及跨平台和小巧客户端的支持以及与你的其它flash应用方案的无缝连接集成。 当然，你需要在培养自已能力所花的精力和你的未来之间做个选择。

多媒体娱乐：在这方面的内容我想我已不用再多说什么。尽管它的发展速度没有像当初预言的那样迅速（就像计算机那样），但它仍然还在不断前进，全新功能也正在扩大它的应用范围。使用这一点无疑很优秀，并且可以为你节约大量的成本。Flash本身就以多媒体和可交互性而广为推崇。它所带来亲切氛围相信你的每一位用户都会喜欢。

教学系统：要在教学系统应用Flash现有的技术无疑会极大增强学生的主动性和积极发明的能力。在这方面涵盖的内容，可能所有现有技术你都可以用上了。根据你教学内容的不同你可能会选择不同的部份。

压缩技术的开发： 是将数据以特定的格式存储在哪一端这的确有点难以得出结论。如果我向你传递了一个“1”的信息，它代表着一个包含很多信息（如10万字）的信号。而这些信息在你那边已经拥有了，你只需要将这些与“1“相对应翻出来就行（解压）。而如果你没有它。那么我不得不向你传送这10万字的信息，当如果这种情况发生在多次，你很快就能明白它的含义了。就像在现实生活中你向你的好伙伴眨眼示意那样。这需要“心有灵犀一点通”，而在这当中，这就是压缩/解压方案。

基于安全封闭网络的视频会议开发：这很有趣，至少和MSN中的视频会议比起来。和MSN中的视频会议中的便于使用比起来，你能在这里找到更多的乐趣和自主能力。

在线娱乐系统：在线娱乐显而易见的要求你有更多更好的作品。如果没有，即使你的系统规划得多么完整或者优秀你也只能将它卖给别人了。我只是要告诉你你也应该花些时间在你的可供观赏的作品的选择和制作上以及和你的游览者打成一片。至于系统的细节。我想一群优秀的flasher和后台人员会解决它。你所遇到的问题几乎是基于Web服务方面的问题。

及MTV点播系统：我不知道是应该谁最先开始这方面的努力，事实上，这篇教程中的大部份就是在向你讲解关于这方面的知识。我所要告诉你的是：如果你想要一个跨平台的系统，那么就需要付出更多的努力。（当然，如果只是在计算机上，这并不难）

生物技术的模拟：很多人都在这方面尝试努力着。在网上你也可以找到很多的相关资源。对于这方面的内容更重要是的你的其它学科（如数学、生物学、物理学...）的内容，你只是在flash中模拟它们并提供双向的交互能力。以下是其中的一些试验站点： levitated（拥有众多的开放源代码，其中也包括对生物运动和自然形态过程的模拟）http://www.levitated.net/ uncontrol（同样不错的站点）　　http://www.uncontrol.com/

仿真及虚拟视觉：仿真的3D世界可能在现在这个时候你要付出更加艰辛的努力，因为Flash中对3D的引擎支持得不多。需要你自已更多的数学计算思维和你的空间想像。基于红-蓝双眼模拟可能会让你更容易些，不过这需要3D眼镜的支持。

你的BBS论坛系统：事实上大部份使用在这方面功能的flasher都选择了像以前的HTML的样式那样建立他们的全Flash论坛系统。但也并意味你必须跟从他们。可能他们这样做是出于技术和习惯上的考虑，这差不多只不过将HTML翻了一个版而已。它们丢失了太多的Flash中激动人心的地方。当然，创造意味着从你自已开始。相信在Flash Player7中对CSS和HTML文本部份内嵌图像和swf的支持会让你更加得心应手。

当然，还远不止这些，你所想要的项目应该由你自已来发现和挖掘。你可以到网上查找到更多的相关信息和功能。Macromedia中国的站点也很值得一看。
