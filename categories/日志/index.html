我的设计博客

为了我的设计博客，前几天建的Lofter博客，今天终于布置好了。

不得不说，网易的Lofter轻博客做得很好，各种自定义功能与模块,管理也很方便快捷。其中博客模板就有很多，而且很漂亮，不过不能随意修改，只能通过其设定好的自定义项修改。

今天挑来挑去，不是差这个不够理想，就是差那个不够美观。最后，终于确定了一个，响应式设计的，没有什么重大缺点也没有什么亮点的模板。

博客名字叫伴星流形，自己还是有点文学情节，起了这个名字，伴星是我一直以来用的网名，流形是源于文天祥的《正气歌》：

天地有正气，杂然赋流形。下则为河岳，上则为日星。

另外还有易经《彖》：

大哉乾元，万物资始，乃统天。云行雨施，品物流形。

流形，字面上解释起来就是：各种形态，指天地宇宙间的一切形体。

一经看到这个词，我就喜欢上了它了，流形流形，设计不就是这样吗？上善若水，大道无边。任何领域从技术升到艺术层面就应该不再有约束存在了，这和我高一的语文老师说的差不多，原话忘了，大致意思是：有些同学，写的作业以为自己在写狂草，字难看得我看都不想看。你们现在写字就要一个一个写工整，等你达到更高境界了，才能随心所欲，任笔挥写，讲求意到笔随。

所以博客名叫伴星流形，英文名取流形翻译叫Manifold。愿我的设计能有星空的虚空磅礴，亦有流形般不受约束，尽情绽放自己的创想。

2013年小小总结

又一年了，小小总结下。

在12年1月正式学习flash开发，13年1月还是那么满怀期待的尝试flash新技术（starling、增强现实），14年1月了，我开始认真关注设计圈子。

是的，我“转行”了。

严格上说不叫转行，只是我将重心天平大大的移到设计上。一直以来对设计的兴趣没有让我落下。零零碎碎的，只是缺少能拿的出手的作品。

2013年的历程：

  • 3月开始成为工作室负责人之一。与庆杰一起带新一届的维生数。
  • 4月策划带人做手游Pulupulu（目前停留于那时完成的测试版）
  • 5月参加学校社团组织的平面动画设计大赛（动画一等）
  • 7月暑期实践学习android开发新生网app（失败）
  • 10月开发桌面应用GIMS（目前版本v0.6.5）
  • 11月带队参加“规划杯”比赛（优秀奖）

特别的感受呢，有三方面：做工作室负责人，带项目，参加比赛。

做工作室负责人。我和庆杰带的这届，到现在做得也算是足够了。刚接手时，那几乎是工作室的最低潮点，服务器垮了之后焦头烂额没弄好，交给了我们。那时，服务器坏了，工作室主页不在了，博客没了，logo丢了原图，工作室缺少了载体，人也散了。从今年3月接手以来，重新建立博客、认真举办每次分享会，购置投影仪看电影，开总结会议、谈未来计划，解决服务器问题，开各种交流会，做了几个自主项目，招新也做得有范了。在不停地处理工作室各种事务、技术完善与追求中，逐渐形成了一套“独特”的管理、学习、交流方法。

但，问题还是有的，几个项目“烂尾”，以及工作室这近乎华丽的外表上，似乎预示着工作室正在崛起的路上，实则不然，细心发现，就会觉得有失初衷——大家玩在一起。上学期期末记得潇滨提醒过我们：技术是很好了，但活动不能少。

执行力与凝聚力是一直存在的问题，执行力很大方面是项目组里面需要有一个好leader。而工作室很少组织活动，因为一旦组织，难有集体响应，只能几个小团体间的活动。在工作室煮煮糖水、煮煮咖喱饭啊，真的很好吃~但才那么几次。工作室嘛，大家就要玩得起来才叫工作室~

这一年来我们追求技术太过了，太过于技术这块，而让我们在这路上丢失了很多其他东西。这之后需要我们再度拾起。

带项目 发现我的大局观越来越好了，可能是我兴趣广泛让我的眼界不断开拓，以及学会了一套适合自己的学习方法，似乎一切技术上的问题都难不倒我，“似乎”。

这一年带了三个算是项目的项目，一是手游pulupulu，4人，一个月。这算是一次带新人吧，我做开发，但我会查阅学习策划、设计、开发实现，我尝试了之后再简单地教他们策划、设计、开发。这让我发现了一旦你对这个项目的整体及各方面都掌握到你一个人只要有时间都能独立完成的程度，你是一个合格的项目负责人了，因为你对这个项目了如指掌，能够很好地确立计划与ToDo List，一步一步推进项目进度。

二是新生网App，4人，数天，我做设计和协同开发，在暑假实践要求几天完成，显然不可能，大家都懒惰得后面回家都没去完成了。烂尾！

三是GIMS，这是一次自己的独立开发，将近一个月时间。设计师给了设计稿后交流了几次后开始工作。

总结一下，项目需要一个好Leader，去爱这个项目、产品，也带着大家一起去爱。这样项目才会充满生机与活力。同时负责人需要有良好的大局观，做好规划，分配好工作，并随时应对各种变化。这样才能较好地完成一个项目。

参加比赛 一是社团那个平面设计动画大赛的动画组，几天完成了一个小动画也能拿奖，只能说明我是真的切中了要点，迎合了评委的心意。一是设计上算是创新了，二是展示也较好，三是主题鲜明积极向上。

二是规划杯这个比赛，5人，花了两个星期。这是一次策划比赛，技术含量不高，但我用带项目的管理经验去带队参加这个比赛，效果不错。规划、线上讨论、分工、进度跟进，一步一步完成下来，本以为三等前无误，结果只拿了个优秀奖，不过奖金也有4K。

参加比赛这件事，我算是较有心得的，凡是获奖机会不大的比赛我是不会参加的，(难道参加比赛只为了获奖吗？确实，细心想想你就会发现参加比赛你获得的进步并不会太大，主要是图个参与，顺带获个奖不更好吗？）而参加比赛我一定会死死抓住获奖关键点，我会去想主办方需要得到的是什么作品？评委需要看到的是什么作品？其它关系不大的事情不会太花时间与精力。哲学上这叫“主要矛盾”，策划比赛你去抠设计，设计比赛你去抓狂代码，这不是笑话吗？

过去一年平平淡淡，没有跌宕起伏经历，却也有希望与失望，有快乐与伤心，有成功与失败。

确实让我蜕变了很多，至少锐气磨减了，眼界更开阔了，看的书多了，技术也进步了许多。

嗯，一切安好，还需努力，2014年继续加油！晚安。

博客搬家——也算是个转折点

事情要从这个月15日说起，那时去上很久没写过的博客，发现已经挂了，坑爹的免费主机商，差点以为所有文章都没了，还好有备份数据库。早就在暑假时就要将博客搬到jekyll了，不过在windows平台ruby不方便。前阵子交流会上听说hexo，看上去不错，采用Node.js,上学期就用过Node.js,搭起来很快捷。找个免费主机把数据库恢复，再从wordpress后台导出xml文章数据，将此转为markdown格式，安装相关插件：

 $ npm install hexo-migrator-wordpress —save

再运行命令（为刚才的导出的xml文件地址）:

$ hexo migrate wordpress < source >

最后获得的md格式的文章。generate后将public文件夹的内容pull上github page就好了，暂时用着默认主题先，打算什么时候自己做个博客主题玩玩。

转折点

用了快两年的wordpress博客，终于是要退役了啊。

一眼扫过，看到自己一年前还是个小白，以及青涩的口语，现在是很难再启齿在博文里了吧。想删去一些“无聊”的博文的，不过转念一想还是算了吧，算是为我的青春留个纪念。岁月果然无情，被催赶着踏在不能不成熟的道路上，原理稚气与懦弱，走向沉稳与成熟。到现在为止，我走上flash这条道路一年半多了，回想起来，真是感慨，当时还很满意自己成为了一个flasher，如今这个称号，只能呵呵了。走flash这条路，真的是一条路走到黑啊，从最开始玩简单的as3代码，到接触air开发、flex开发，再到上半年的Starling，flash总能给我太多惊喜，开发速度快，效率高，效果好，性能也还说得过去。但flash却还是要逐渐退出舞台，身边的人在做前端、玩html5、安卓开发，我也常常心动，暑假还玩了会android开发，期间越来越觉得自己对flash、as3情有独钟，怎么写怎么不合适，想用as3写，哈。

总结自己，设计方面，从小就有绘画情怀的我，爱设计，但也喜欢编程，一直不忘设计，可以说，我的设计感觉、眼界一直在培养，看设计、交互方面的书，有空就看dribbble上的优秀作品，前阵子开始话icon，明显感觉到自己的设计水平在进步。编程方面，主攻flash开发。前端一般般，能看，但没正式写过一个网站；后台也只是用过php数据库，并不太精通，这阵子做air应用程序还要学用SQL数据库；安卓开发也只是半桶水。

计划

也快到学期中了，开学家里一些事，影响了下。后来慢慢回归正常，每周参与工作室的“设计交流会”，“前端-后台交流会”；闲时看看设计，做做设计舒缓舒缓心情；上周末开始尝试用Air+Morn UI做GIMS这个信息管理应用程序（软件）；还有看书，阅读。后面打算，继续画icon，等画得差不多就做app UI，再做网站设计；做完这个GIMS软件；学好flash设计模式及后台，完成上学期烂尾的手游；完成暑假没完成的安卓应用；寒假学前端，写一个网站。

显然我不是天才，我编程有时会很苦恼，我设计有时会很渣，但如一句说的，

“你必须十分努力，才能表现的毫不费力。”

下学期就要去找实习了！加油！

知其然而不知其所以然

我感觉我现在对AS3的学习就是这样，这几天在做个仿切水果的游戏常常遇到问题经过不断尝试解决了后，却很难说出其中的原理。

像昨天忙了一个晚上对一个变量的引用，才搞懂问题所在；以及今天下午尝试了很久才发现添加enterFrame事件侦听器的地方不该在那里；以及对数组的引用。虽然在尝试解决的过程中有了新的认识，对一些代码有更深的了解，但我那感觉还是模棱两可的，因为自己都不敢确定是不是所想的那样。再次遇到同样的问题，却找不到根本的原因，得到最直接的解决办法，而又需要不断地尝试，才会发现另一个解决办法，而还是不知道其中的真正原因。

显然这对于一个代码学习者来说是不好的，虽然这样学会的会印象深刻，但却走了很多弯路，也浪费了很多时间。本身大学空余的时间就不多，我不该在这方面去浪费时间。而且我也感觉我是太过急了，不好好学基础，把基础打劳，就去实践。没完全搞明白化学反应原理，就去搞化学反应，这是同样的道理。

打字 日志

打字那些事儿

打字一直是我很向往的事情，当然是指那些能五指如飞般哒哒哒的打字，以前每当看到这种人都会很羡慕。我也很想学，可是一直都学不会那么快的打字，也没有按照五指摆正的打字，还是手指怎么习惯怎么快就怎么敲，这是我心中永远的痛啊……

但是现在学代码了，深感打字的重要性，不然打代码效率大大折扣。前短时间又重新安装了金山打字通，没错，就是金山打字通，小学就开始用了，现在用起来依然是那么好用，于是就开始了我的打字之旅。

每天晚上都要打上个把小时，打开音乐，戴上耳机，打开金山打字通，边打字边听歌，feel还是不错的哦！而且效果很明显，现在已经有点感觉了，打字也越来越快了。

说说我的经验吧，

1、习惯很重要，必须时刻保持良好的打字习惯，纠正坏的习惯，熟悉各个手指所负责的键位，并且不要随心所欲“越位”。

2、打字就像背单词，背得对自然开心呢，但是总会有那么几个单词经常被错，所以要不断重复，才会印象深刻。

3、必须要有耐心，打字的时间加入没有耐心会觉得时间过得很慢，于是就会想着玩玩游戏上上网算了。但是一旦沉浸在打字的海洋里就会觉得时间过得很快很充实。

4、可以找一些目标追求，例如我时常以金山打字通里的打字速度多少每分钟和正确率作追求。

5、一步一步来，先从初级教程开始打，一个一个教程来，一般每次打三个教程为宜。

6、最后，打累了可以去玩玩这个html5开发的很不错的英文单词打字游戏《Z-Type》 传送门 。

这两天的学习感想

1、发现原来舞台stage.stageWidth和stage.widht是不同的，前者是真正的舞台的宽，后者是舞台内所有元件的宽；原来坐标分全局坐标和本地坐标；还有注册点和中心点；原来直角坐标系又叫笛卡尔坐标系……在学习尝试中我发现了很多以前没注意到的细节和知识，也注意到了自己经常范一些错误，例如常把事件类型等前的“：”打成了“.”号，也常常把“.”打成了“,”号，还有时会把一些英文打错了，这些小错误经常都是测试后仔细找寻才发现的。

我想这些错误是因为我还不熟悉AS3，所以要经常接触，才不会生疏，即使是寒假实践完后，也不能扔到一边。

2、学习了数学基础和物理基础，觉得数学知识在flash里应用很广，三角函数、角度、直线、图形、向量等，很多地方都要用到。物理知识中速度、加速度、碰撞、向量等的计算和应用也很有用，虽然都是基础，但掌握之后是很有用的。终于觉得初中和高中学的数学物理有点用处了！汗。

3、由于flash基于帧的，常常是以帧频来作用函数的，反应会慢，当物体速度过大时，也会出现很多问题，例如穿墙（碰撞）、显示跳动等，但也要有一些解决方法:加上提前检测代码，改用其他方法解决跳动等。我想现在接触还浅到，以后会出现更多问题，有时也会让人焦头烂额，但我已经做好准备了。

4、面向对象编程，这应该是很多编程代码都有的算是很基础的一种方法吧，但开始学时我还是觉得很难搞清一些概念，自己照着做了一个小球运动的文件，弄了两个类文件，一个是自建的Ball类，还有一个主要的类（其中会引用Ball建一个小球，我以为将类文件和空白fla摆在一起就可以了，但播放时还是没东西显示，原来要在fla属性上设置文档类文件，还要设置类路径，设置Ball.as的位置。在学习flash面向对象的过程中，还是有些没搞清楚，但这急不来吧，以后慢慢来 →_→

5、其实挺期待后面5天的制作的，尤其是最后两天的水果忍者制作，但还是得慢慢来，不能急，但我也在细细准备着了- - -
